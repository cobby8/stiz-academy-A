# Lessons

## 2026-07-11: 관리자 주 본문은 캐시 서버 payload가 client fetch보다 낫다
- 현상: `/admin/programs`, `/admin/schedule`은 관리자 셸이 1초대에 먼저 보였지만, 실제 본문은 클라이언트 API를 기다리느라 약 5초 뒤 표시됐다.
- 원인: 서버 렌더를 줄이려고 본문 데이터를 브라우저 fetch로 미루면, 사용자는 빈 껍데기를 먼저 보고 실제 업무 화면을 다시 기다리게 된다.
- 해결: 선생님이 진입 즉시 봐야 하는 주 본문 데이터는 `unstable_cache`로 짧게 캐시한 서버 payload를 페이지 첫 렌더링에 주입한다.
- 예방: “자주 쓰는 목록/시간표/CRM”은 첫 화면에 싣고, “수강생 선택 목록/설정/모달 내부 보조 데이터”처럼 필요할 때만 보는 데이터만 뒤로 미룬다.

## 2026-07-06: 문서보다 코드가 더 최신일 수 있다
- 개발 기록 문서는 2026-03-20 기준 내용이 많지만, 실제 최근 커밋은 리치 에디터, 다크모드, 신청 폼, 공지 개선까지 더 진행되어 있었다.
- 앞으로는 문서를 먼저 읽되, 실제 라우트/메뉴/스키마/최근 커밋으로 반드시 대조한다.

## 2026-07-06: 프로젝트 이름만 보고 쇼핑몰로 단정하지 않는다
- 사용자가 “쇼핑몰”이라고 표현해도 실제 저장소는 학원 운영 플랫폼일 수 있다.
- 기능 구조를 먼저 확인한 뒤 현재 프로젝트의 정확한 성격을 맞춘다.
