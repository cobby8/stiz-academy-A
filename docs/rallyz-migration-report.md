## 3. 랠리즈 데이터 마이그레이션 계획

### 3-1. 마이그레이션 대상 데이터

| 데이터 유형 | 랠리즈에서 추출 방법 | STIZ 저장 위치 | 비고 |
|------------|---------------------|---------------|------|
| **원생 명단** | 엑셀/CSV 다운로드 | DB: Student 테이블 | 이름, 생년월일, 학부모 연결 |
| **출결 이력** | 엑셀/CSV 다운로드 | DB: Session + Attendance | 날짜별 출결 기록 |
| **수납 내역** | 엑셀/CSV 다운로드 | DB: Payment 테이블 | 청구/납부 이력 |
| **수업 사진/영상** | 수동 다운로드 필요 | Supabase Storage | ⚠️ 대용량 |
| **공지사항** | 수동 복사 또는 API | DB: 새 테이블 필요 | 텍스트 + 첨부파일 |
| **클래스 정보** | 엑셀/CSV 또는 수동 | DB: Class, Program | 현재 데이터로 대체 가능 |
| **채팅 내역** | 추출 불가능 가능성 높음 | - | 이전 불필요 판단 |

### 3-2. 마이그레이션 일정 (권장)

```
[D-30] 사진/영상 갤러리 기능 개발 완료

[D-14] 알림 시스템 기본 구현 (이메일)
[D-7]  랠리즈에서 모든 데이터 다운로드 (엑셀 + 사진)
[D-5]  데이터 정제 및 변환 스크립트 작성
[D-3]  테스트 환경에서 마이그레이션 리허설
[D-1]  최종 데이터 다운로드 (마지막 동기화)
[D-Day] 프로덕션 마이그레이션 실행
[D+1]  검증 및 학부모 안내
[D+7]  랠리즈 해지
```

### 3-3. 필요한 DB 스키마 추가 (마이그레이션용)

```prisma
// 공지사항
model Notice {
  id        String   @id @default(uuid())
  title     String
  content   String
  classId   String?  // null이면 전체 공지
  authorId  String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  class     Class?   @relation(fields: [classId], references: [id])
  author    User     @relation(fields: [authorId], references: [id])
  attachments NoticeAttachment[]
}

// 공지 첨부파일
model NoticeAttachment {
  id       String @id @default(uuid())
  noticeId String
  fileUrl  String
  fileName String
  fileSize Int
  notice   Notice @relation(fields: [noticeId], references: [id])
}

// 사진/영상 갤러리
model GalleryPost {
  id        String   @id @default(uuid())
  title     String
  content   String?
  classId   String?  // null이면 전체 공개
  authorId  String
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  class     Class?   @relation(fields: [classId], references: [id])
  author    User     @relation(fields: [authorId], references: [id])
  media     GalleryMedia[]
}

// 갤러리 미디어 (사진/영상)
model GalleryMedia {
  id        String @id @default(uuid())
  postId    String
  mediaUrl  String
  mediaType String // "image" | "video"
  fileSize  Int    // bytes
  post      GalleryPost @relation(fields: [postId], references: [id])
}

// 학습 피드백
model StudentFeedback {
  id        String   @id @default(uuid())
  studentId String
  coachId   String
  sessionId String?
  content   String
  createdAt DateTime @default(now())
  student   Student  @relation(fields: [studentId], references: [id])
  coach     Coach    @relation(fields: [coachId], references: [id])
}
```

## 4. DB 데이터 사용량 및 비용 예측

### 4-1. 현재 Supabase Free Tier 한도

| 리소스 | Free Tier 한도 | 현재 예상 사용량 |
|--------|---------------|-----------------|
| DB 용량 | 500 MB | ~10 MB (텍스트 데이터만) |
| File Storage | 1 GB | ~5 MB (코치 이미지 몇 장) |
| Storage 전송량 | 2 GB/월 | 미미 |
| DB 전송량 | 2 GB/월 | 미미 |
| MAU (인증) | 50,000 | ~50명 미만 |

### 4-2. 사진/영상 업로드 시 예상 사용량

#### 가정
- 원생 수: **50명**
- 주당 수업: **5일**
- 수업당 사진: **평균 5~10장** (약 3MB/장, 압축 후 ~500KB/장)
- 월간 영상: **2~4개** (약 50MB/개, 압축 후 ~20MB/개)

#### 월간 스토리지 증가량

| 항목 | 계산 | 월간 증가량 |
|------|------|------------|
| 수업 사진 | 5일 × 4주 × 8장 × 500KB | **~80 MB/월** |
| 수업 영상 | 3개 × 20MB | **~60 MB/월** |
| 공지 첨부파일 | ~5개 × 500KB | **~2.5 MB/월** |
| **합계** | | **~142.5 MB/월** |

#### 연간 누적

| 기간 | 누적 스토리지 | Free Tier (1GB) 초과? |
|------|-------------|---------------------|
| 1개월 | ~143 MB | ❌ 여유 |
| 3개월 | ~428 MB | ❌ 여유 |
| 6개월 | ~855 MB | ⚠️ 근접 |
| **7개월** | **~998 MB** | **🔴 초과 시점** |
| 12개월 | ~1.7 GB | 🔴 초과 |

### 4-3. Supabase 요금제별 비용 비교

#### Free Tier (현재)
- **월 비용**: $0
- **한계**: Storage 1GB, DB 500MB, 프로젝트 7일 미사용 시 일시정지
- **적합 기간**: 사진/영상 업로드 시작 후 **약 6개월**

#### Pro Tier (권장)
- **월 기본 비용**: **$25/월** (~₩33,000)
- **포함량**: DB 8GB, Storage 100GB, 전송량 250GB
- **초과 요금**:
  - DB: $0.125/GB
  - Storage: $0.021/GB
  - 전송량: $0.09/GB

#### 예상 월간 비용 (Pro 기준)

| 시나리오 | DB | Storage | 전송 | 월 비용 |
|---------|-----|---------|------|--------|
| **현재 (텍스트만)** | ~10MB | ~5MB | 미미 | $25 (~₩33,000) |
| **사진 업로드 시작 후 6개월** | ~50MB | ~1GB | ~5GB | $25 (~₩33,000) |
| **사진+영상 1년 후** | ~100MB | ~2GB | ~10GB | $25 (~₩33,000) |
| **사진+영상 2년 후** | ~200MB | ~4GB | ~15GB | $25 (~₩33,000) |
| **사진+영상 5년 후** | ~500MB | ~9GB | ~20GB | $25 (~₩33,000) |

> **결론**: Pro 요금제의 100GB Storage 한도 내에서 **5년 이상 추가 요금 없이** 사용 가능.

### 4-4. 비용 최적화 전략

1. **이미지 압축**: 업로드 시 자동 리사이즈 (최대 1920px, WebP 변환) → 용량 70~80% 절감
2. **영상 압축**: 720p 이하로 트랜스코딩, 또는 YouTube 비공개 링크 활용 → Storage 절약
3. **오래된 미디어 아카이빙**: 1년 이상 된 사진은 저가 스토리지로 이동 또는 삭제
4. **썸네일 생성**: 목록에서는 썸네일만 로드, 원본은 클릭 시 로드

## 5. 종합 권장사항

### 즉시 실행 가능
1. ✅ 랠리즈에서 원생 명단, 출결 이력, 수납 내역 **엑셀 다운로드** (데이터 확보)
2. ✅ 랠리즈의 수업 사진을 **수동 다운로드** 시작 (시간 소요됨)

### 개발 우선순위
1. **사진/영상 갤러리** — 학부모 만족도에 가장 큰 영향
2. **공지사항** — 소통 채널 확보
3. **알림 (이메일)** — 출결/공지 자동 알림
4. **PWA 설정** — 앱 설치 없이 모바일에서 앱처럼 사용

### 요금제 전환 시점
- 사진/영상 기능 **개발 완료 후** Pro 전환 권장 ($25/월)
- Free Tier에서 먼저 개발하고, 실제 미디어 업로드 시작 시 전환

### 랠리즈 해지 전 체크리스트
- [ ] 모든 원생 데이터 엑셀 다운로드 완료
- [ ] 모든 수업 사진/영상 다운로드 완료
- [ ] 출결 이력 전체 다운로드 완료
- [ ] 수납 내역 전체 다운로드 완료
- [ ] STIZ 시스템에 데이터 마이그레이션 완료
- [ ] 학부모 전원에게 전환 안내 완료
- [ ] STIZ 시스템에서 학부모 로그인/마이페이지 정상 작동 확인
