## 구현 기록
- `src/app/actions/admin.ts`
  - 활성 `BillingTemplate`과 ACTIVE 수강생을 조합해 월별 청구 대상 미리보기를 계산하는 `previewMonthlyInvoices` 추가.
  - 월별 청구서 생성은 루프 대신 DB `INSERT ... SELECT`로 일괄 처리.
  - 기존 같은 학생/연월/청구 유형 결제는 유지하고 중복 생성하지 않도록 분리.
  - 청구서 생성 시 학부모 SMS 자동 발송을 제거하고 미납 알림 버튼과 분리.
- `src/app/admin/finance/FinanceClient.tsx`
  - 상단 버튼을 `청구 대상 확인`으로 변경.
  - 월별 청구 대상 카드에 활성 템플릿, 대상 학생, 생성 예정, 기존 유지 건수/금액 표시.
  - 미리보기 카드 안에서만 `청구서 생성` 버튼을 노출.
