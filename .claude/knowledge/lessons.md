### [2026-03-26] 가이드 투어: 직접 구현 vs 라이브러리 판단 기준
- **분류**: lesson
- **발견자**: planner-architect
- **내용**: "구현 범위가 작으니까 직접 만들자"는 판단이 5회 실패로 이어졌다. CSS 오버레이/하이라이트는 단순해 보이지만, 브라우저의 stacking context/overflow/pointer-events 조합이 예측 불가능한 함정을 만든다. 판단 기준: (1) "특정 DOM 요소를 시각적으로 분리"하는 기능은 SVG clipPath가 유일하게 안정적, (2) SVG clipPath 직접 구현은 driver.js 수준의 복잡도 -> 차라리 라이브러리 사용, (3) 라이브러리가 처리 못하는 부분(페이지 간 이동)만 자체 구현하는 것이 최적.
- **참조횟수**: 0
