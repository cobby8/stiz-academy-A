### 조사 결과: 외부 플랫폼 리뷰 수집 방법 (2026-03-26)

#### 1. 네이버 플레이스 리뷰

**공식 API: 없음**
- 네이버 오픈 API 목록에 플레이스 리뷰 API는 존재하지 않음
- 지역 검색 API(/v1/search/local)로 업체 기본 정보(이름, 주소, 카테고리)는 조회 가능하나 리뷰 데이터는 미포함
- 2026년 4월 6일부터 네이버 플레이스에 5점 별점 제도 도입 예정 (기존 키워드 리뷰에 추가)

**비공식 크롤링: 기술적으로 가능하나 위험**
- 기술적: 네이버 플레이스 페이지를 스크래핑하여 리뷰 수집 가능 (AI Canvas 등 3rd party 도구 존재)
- 법적 리스크: 높음. 네이버 DB권 침해 + 이용약관 위반 가능. 네이버는 무단 크롤링에 적극적으로 법적 대응
- 기술적 리스크: 네이버가 구조 변경 시 즉시 깨짐. 유지보수 부담 큼
- **추천: 비추천**

**임베드/위젯: 없음**
- 네이버 플레이스 리뷰를 iframe으로 임베드하는 공식 방법은 없음
- 네이버 지도 임베드는 가능하나 리뷰 섹션은 포함 안 됨

| 항목 | 결과 |
|------|------|
| 기술적 가능 | 크롤링만 가능 (공식 API 없음) |
| 비용 | 무료 (크롤링) |
| 법적 리스크 | 높음 (DB권 침해, 약관 위반) |
| 구현 난이도 | 높음 (스크래핑 + 유지보수) |
| 추천 | 비추천 |

#### 2. 구글 플레이스 리뷰

**공식 API: 있음 (Google Places API)**
- Place Details API로 리뷰 데이터 조회 가능
- 가져올 수 있는 데이터: 별점(rating), 리뷰 텍스트, 작성일, 작성자 이름, 프로필 사진, Google Maps 프로필 링크, 언어 코드
- 제한: 최대 5개 리뷰만 반환 (최신/관련순). 전체 리뷰 목록 가져오기 불가

**비용 (2025년 3월 이후 새 가격 체계)**
- reviews 필드는 Pro SKU에 해당
- Pro: 월 5,000건 무료 / 초과 시 $17 / 1,000건
- 학원 사이트 트래픽이면 월 5,000건 무료 범위 내 충분할 가능성 높음
- 단, Google Cloud 프로젝트 + 결제 계정 설정 필요 (신용카드 등록)

**위젯/임베드: 풍부한 선택지**
- 무료 3rd party 위젯 다수 존재:
  - EmbedSocial: 무료 플랜 있음 (브랜딩 포함)
  - Elfsight: 무료 위젯 (72시간 내 자동 동기화)
  - SociableKIT: 무료, 코딩 불필요, HTML embed 코드
  - Shapo: 무료 플랜 (리뷰 10개까지)
  - Jotform: 무료, Google Business Profile 자동 동기화
- 대부분 Google Business Profile 연결 -> 코드 한 줄 붙여넣기로 완료

| 항목 | 결과 |
|------|------|
| 기술적 가능 | 가능 (공식 API + 무료 위젯 다수) |
| 비용 | 무료~저렴 (위젯 무료, API 월 5,000건 무료) |
| 법적 리스크 | 없음 (공식 API/위젯) |
| 구현 난이도 | 낮음 (위젯) ~ 중간 (API 직접 연동) |
| 추천 | 강력 추천 (위젯 방식) |

#### 3. 카카오맵 리뷰

**공식 API: 없음**
- 카카오맵 API는 장소 검색(이름, 주소, 좌표, 카테고리)만 제공
- 리뷰/평점 데이터는 API로 제공하지 않음 (카카오 데브톡에서 공식 확인)
- 크롤링도 네이버와 동일한 법적/기술적 리스크

| 항목 | 결과 |
|------|------|
| 기술적 가능 | 불가 (API 미제공, 크롤링만 가능) |
| 비용 | - |
| 법적 리스크 | 높음 (크롤링 시) |
| 구현 난이도 | - |
| 추천 | 비추천 |

#### 5. 우리 프로젝트 적용 방안 (추천 순서)

**방안 A: 구글 리뷰 무료 위젯 (최추천, 난이도 하)**
- SociableKIT / EmbedSocial / Elfsight 중 택 1
- Google Business Profile 연결 -> embed 코드 한 줄 붙여넣기
- 자동 동기화되어 새 리뷰 자동 반영
- 구현: 30분 이내
- 단점: 위젯 서비스의 무료 플랜은 보통 브랜딩(로고) 포함

**방안 B: 구글 리뷰 API 직접 연동 (난이도 중)**
- Google Places API Place Details로 최신 리뷰 5개 가져오기
- Next.js ISR로 캐싱 (revalidate: 3600 등)
- 디자인 자유도 높음 (우리 사이트 스타일에 맞춤)
- 단점: 최대 5개만 가져올 수 있음. Google Cloud 결제 계정 필요

**방안 C: 관리자 수동 등록 (현재 구조 개선, 난이도 중)**
- 현재 하드코딩된 TestimonialCarousel을 DB 기반으로 전환
- 관리자가 네이버/구글 리뷰를 직접 복사해서 등록
- 장점: 외부 의존 없음, 리뷰 선별 가능
- 단점: 수동 작업 필요

**방안 D: 하이브리드 (A+C, 가장 이상적)**
- 랜딩 페이지: 관리자가 선별한 베스트 후기 (방안 C)
- /reviews 페이지 또는 하단: 구글 리뷰 위젯 (방안 A)
